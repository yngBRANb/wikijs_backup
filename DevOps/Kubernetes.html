CI/CD

Вариант 1: GitOps-подход (рекомендуется для Kubernetes)

Для CD будет использоваться Argo CD (можно так же и Flux CD), а для CI требуется выбрать внешнюю систему на свое усмотрение:

  • CI (сборка и тесты): Jenkins / GitHub Actions / GitLab CI
  • CD (деплой): Argo CD

Вариант 2: Классический CI/CD

Предполагает внедрение агента в K8s:

  • Jenkins
  • GitLab CI
  • GitHub Actions
  • Jenkins X (для облачной среды)

Вариант 3: Специфичные подходы

  • Werf
  • Werf + Argo CD (имеет преимущества GitOps)
