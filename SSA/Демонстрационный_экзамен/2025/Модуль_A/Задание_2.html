Настройте имена устройств согласно топологии. Используйте полное доменное имя:

По заданию необходимо задать полное доменное имя устройства. Необходимые имена указаны в Таблице:

Для изменения имени на Linux устройстве необходимо редактировать файл hostname:

vim /etc/hostname

В случае с Linux, чтобы hostname применился нужно перезагрузить устройство.

Для изменения имени на Eltex (в данном примере HQ-RTR) необходимо выполнить следующие команды:

vesr# config
vesr(config)# hostname hq-rtr.au-team.irpo
vesr(config)# exit
vesr# commit
vesr# confirm

На всех устройствах необходимо сконфигурировать IPv4

Процесс настройки сети на ALT Linux смотрите здесь

Процесс настройки сети на Eltex:

HQ-RTR:

# Узнайте по MAC какой интерфейс куда смотрит
HQ-RTR# show interfaces status

HQ-RTR# config
HQ-RTR(config)# int gi 1/0/1
HQ-RTR(config-if-gi)# description external
HQ-RTR(config-if-gi)# ip firewall disable
HQ-RTR(config-if-gi)# ip address 172.16.4.2/28
HQ-RTR(config-if-gi)# exit

HQ-RTR(config)# interface gi 1/0/2
HQ-RTR(config-sub-in)# description servers
HQ-RTR(config-sub-in)# ip firewall disable
HQ-RTR(config-sub-in)# ip address 192.168.10.1/26
HQ-RTR(config-if-gi)# exit

HQ-RTR(config)# interface gi 1/0/3
HQ-RTR(config-sub-in)# description clients
HQ-RTR(config-sub-in)# ip firewall disable
HQ-RTR(config-sub-in)# ip address 192.168.10.65/28
HQ-RTR(config-if-gi)# exit

HQ-RTR(config)# interface gi 1/0/4
HQ-RTR(config-sub-in)# description mgmt
HQ-RTR(config-sub-in)# ip firewall disable
HQ-RTR(config-sub-in)# ip address 192.168.10.81/29
HQ-RTR(config-if-gi)# exit

HQ-RTR(config)# ip route 0.0.0.0/0 172.16.4.1
HQ-RTR(config)# end
HQ-RTR# commit
HQ-RTR# confirm
