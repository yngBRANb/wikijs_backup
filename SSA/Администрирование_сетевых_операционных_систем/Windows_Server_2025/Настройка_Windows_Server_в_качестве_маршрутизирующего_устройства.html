Цель работы: настроить Windows Server 2025 на маршрутизацию пакетов и трансляцию адресов (NAT).

Краткие теоретические и учебно-методические материалы по теме лабораторной работы:

На практике маршрутизаторы на базе компьютеров используются довольно редко (аппаратные маршрутизаторы, как правило, имеют более высокую производительность, надежность и несколько дешевле выделенного компьютера), в тестовых или виртуальных средах, когда нужно срочно настроить маршрутизацию между несколькими подсетями, маршрутизатор на базе Windows Server вполне себе приемлемое решение.

Так же бывают случаи когда стоит задача разделить сеть на два сегмента, при этом без закупки маршрутизатора. На него могут не выделить деньги, такое можно встретить в компаниях из гос. сектора. И в качестве решения этой проблемы можно рассмотреть настройку компьютера (или сервера) в качестве маршрутизатора.

Задания для практической работы:

Маршрутизация в Windows Server реализуется на базе роли Routing and Remote Access Service (RRAS). Данная служба появилась еще в Windows Server 2003 и до текущей в версии Windows Server ее интерфейс и процесс настройки практически не изменился.

В первую очередь нам нужно установить роль RRAS, а затем ее настроить включив трансляцию адресов (NAT).

1. Установка роли RRAS 

Для этого откроем консоль Диспетчер серверов, выбираем Управление -> Добавить роли и компоненты, находим и отмечаем роль Удаленный доступ, в ее составе выбираем службу Routing, и, соглашаясь со всеми предложенными по умолчанию компонентами, запускаем ее установку (Install).

После того как мы выберем службу ролей Маршрутизация, у нас автоматически выделится роль DirectAccess и VPN (RAS), а так же будет установлена роль IIS (она нам в будущем пригодится)

2. Настройка роли RRAS

После установки у вас появится предупреждение о необходимости настройки роли DirectAccess и VPN (RAS), отройте и закройте мастер настройки, чтобы скрыть его (так как нам не нужна эта роль).

Настройка будет осуществляться через оснастку Маршрутизация и удаленный доступ.

Служба маршрутизации и удаленного доступа позволяет настраивать следующие элементы:

  • безопасное подключение между двумя частными сетями;
  • шлюз виртуальной частной сети (VPN);
  • сервер удаленного доступа;
  • преобразование сетевых адресов (NAT);
  • средства маршрутизации локальной сети;
  • основной брандмауэр.

Перейдем к настройке маршрутизации, для этого отройте утилиту Маршрутизация и удаленный доступ, это можно сделать через Диспетчер серверов:

или через утилиту Выполнить (Win+R), введя название rrasmgmt.msc.

Далее щелкаем по имени сервера (с красной стрелкой) и выбираем Настроить и включить маршрутизацию и удаленный доступ.

На следующем шаге (Подключение к Интернету на основе NAT) нужно выбрать сетевой интерфейс, подключённый ко внешней сети / Интернету (в моем примере это Ethernet0 с адресом 192.168.14.100). Этот интерфейс будет «публичным интерфейсом» нашего NAT роутера.

Нажмите “Готово” после чего дождитесь применения настроек. В процессе может возникнуть ошибка, не обращайте на нее внимание.

На этом базовая настройка маршрутизации на Windows Server 2025 завершена. Сервер уже должен выполнять маршрутизацию пакетов между двумя подключенными сетями и выполнять трансляцию сетевых адресов (NAT).

Чтобы в этом убедиться, в консоли RRAS откройте свойства сервера. На вкладке Преобразование сетевых адресов (NAT) показано, что IPv4 маршрутизация включена (т.е. пакеты IPv4 будут пересылаться с одной сетевой карты на другую).

Проверить работу маршрутизации можно, выполнить ping или трассировку маршрута (tracert) с WCLI к ресурсу, расположенному во внешней сети или интернете. Эти попытки должны быть успешными.
