Краткие теоретические и учебно-методические материалы по теме лабораторной работы:

GNOME (GNU Network Object Model Environment) — одна из популярных сред рабочего стола в мире Linux. Она постоянно поддерживается и довольно легко интегрируется с аппаратным и программным обеспечением. Стоит отметить тот факт, что Canonical даже приняла GNOME для ОС Ubuntu в качестве среды рабочего стола по умолчанию. Однако надо помнить, что GNOME — не самый простой инструмент. Дизайн довольно сильно отличается от привычного виндового интерфейса, и для начального взаимодействия и простоты использования, вам придётся поколдовать с существующими расширениями. Например, если вам необходима панель задач, как в винде, для приложений или переключения окон, то вы можете попробовать для себя Dash to Panel. Нет смысла дублировать тут туториал по установке, лучше воспользоваться инструкцией разработчика. Но помните, что с большим набором расширений оболочка начинает заметно подъедать системные ресурсы слабомощных машинок. 

Существует большое количество разнообразных графических оболочек для ОС Linux. Одни из самых популярных:

  • GNOME
  • GNOME Flashback
  • KDE Plasma
  • XFCE
  • LXDE
  • Cinnamon
  • Pantheon
  • MATE

Стоит учитывать, что одна и таже графическая оболочка в разных
дистрибутивах может выглядеть абсолютно по-разному.

Для удобной установки GNOME в ALT Linux предусмотрено два
виртуальных пакета:

  • gnome3-default - оптимальная установка, подходящая для большинства пользователей
  • gnome3-minimal - минимальная установка

Задания для лабораторной работы:

1. Откройте терминал и введите apt-get update чтобы обновить пакетную базу:

Далее введите команду apt-get install gnome3-default -y для установки пакета gnome в ALT Linux

После этого у вас начнется загрузка пакета gnome для alt. Дождитесь ее завершения.

2. GNOME нормально запускается только через gdm (GNOME Display Manager), поэтому необходимо переключиться на него. Отключаем текущий display manager:

systemctl disable display-manager

Включаем gdm:

systemctl enable gdm

3. Перезагружаем систему:

reboot

4. После перезагрузки залогиньтесь и введите команду активирующую gdm:

systemctl start gdm

Это запустит графический интерфейс:

Выполните авторизацию под вашим пользователем. Под пользователем root пройти авторизацию в средах рабочего стола не получится!

5. Разрешение экрана будет очень маленьким, для решения этой проблемы нужно установить vm-tools. Для этого откройте терминал: 

Нажмите обзор

Введите в поиске terminal
 

В терминале введите команду для перехода под суперпользователя (root) и введите пароль от него:

su -

Установите пакет open-vm-tools-desktop и добавьте его в автозагрузку:

apt-get install open-vm-tools-desktop
chkconfig vmtoolsd onsystemctl enable --now vmtoolsd

Перезагрузитесь:

reboot

Снова зайдите в режим среды рабочего стола (Используйте это команду всегда когда вам нужна будет графическая оболочка):

systemctl start gdm

Проверьте исчезла ли проблемы с разрешением экрана

6. Проблему с разрешением экрана мы решили, но осталась проблема со сломанным шрифтом в терминале. Это легко починить установкой новых шрифтов. Зайдите в терминал и введите:

#Переходим под суперпользователя
su -
#Устанавливаем шрифты от Microsoft (полный список шрифтов можете #увидеть здесь https://clck.ru/3AGaJQ)
apt-get install fonts-ttf-ms -y
#Обновляем кэш установленных в ОС шрифтов
fc-cache -f -v

После этих операций закройте терминал и откройте его заново, проблема должна решиться. Если проблема осталась перейдите в настройки терминала и выберите необходимый шрифт вручную:

Во вкладке «Основное» можете выбрать Темную тему.

7. Чтобы каждый раз не переходить под суперпользователя root командой su – давайте выдадим вашему пользователю user1 возможность использовать команду sudo (Эта команда дает возможность единоразово выполнить любую операцию из режима суперпользователя). Для этого в терминале введите:

su -
apt-get install sudo -y

Откройте файл sudoers через утилиту visudo используя редактор vim

EDITOR=vim visudo

Промотайте в конец файла и введите строку: 

user1 ALL=(ALL) NOPASSWD: ALL

Осталось добавить этого пользователя в группу wheel: 

usermod -aG wheel user1

Затем закройте терминал и откройте его заново (или просто выполните команду exit, чтобы выйти из режима суперпользователя) и выполните команду sudo apt-get update для проверки, результат должен быть следующим:
